# Contributing to G3W-SUITE Ecosystem

Thank you for your interest in contributing to G3W-SUITE! We welcome contributions to any of our repositories, including the core application, web clients, plugins, deployment tools, and documentation.

## How to Contribute

### Reporting Bugs & Feature Requests
* Check the specific repository's issue tracker to ensure the bug or feature hasn't been reported yet.
* Open a new issue in the relevant repository. Be sure to include clear steps to reproduce the issue, your environment details (G3W-SUITE version, QGIS Server version, OS), and expected vs. actual behavior.

### Submitting Code Changes (Pull Requests)
1. Fork the relevant repository and create your branch from `main` or the current development branch.
2. Ensure any new or modified source code files include the appropriate Mozilla Public License 2.0 (MPL-2.0) header.
3. Write clean, documented code and include unit tests wherever applicable.
4. Open a Pull Request (PR) in the specific repository, clearly describing the changes made and referencing any related issues.

## Review and Acceptance Process
* All submitted PRs will be reviewed by the project's core maintainers.
* Feedback will be provided openly and transparently within the PR comments.
* Final approval and merging depend on passing automated CI/CD tests and technical validation by the maintainers.

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).
